자료구조에서 **큐(Queue)**와 **덱(Deque, Double-Ended Queue)**은 데이터를 선형으로 저장한다는 점에서 비슷해 보이지만, **데이터가 들어오고 나가는 통로의 개수**에서 결정적인 차이가 있습니다.

---

### 1. 큐 (Queue)
큐는 **'선입선출(FIFO, First-In-First-Out)'** 원칙을 따르는 자료구조입니다. 줄을 서서 기다리는 것과 같은 원리입니다.

* **구조:** 한쪽 끝(Rear)에서는 삽입만 일어나고, 반대쪽 끝(Front)에서는 삭제만 일어납니다.
* **주요 연산:** `enqueue`(데이터 추가), `dequeue`(데이터 추출).
* **특징:** 순서대로 처리해야 하는 작업(프로세스 스케줄링, 프린터 출력 대기열 등)에 주로 사용됩니다.

### 2. 덱 (Deque)
덱은 **'양방향 큐'**라고도 불리며, 이름 그대로 양쪽 끝에서 삽입과 삭제가 모두 가능한 자료구조입니다.

* **구조:** 앞(Front)과 뒤(Rear) 양쪽 모두에서 데이터를 넣고 뺄 수 있습니다.
* **주요 연산:** `addFirst`, `addLast`, `removeFirst`, `removeLast`.
* **특징:** 스택(Stack)과 큐(Queue)의 기능을 모두 구현할 수 있을 만큼 유연합니다.

---

### 3. 한눈에 비교하기

| 구분 | 큐 (Queue) | 덱 (Deque) |
| :--- | :--- | :--- |
| **데이터 방향** | 한 방향 (단방향) | 양 방향 (이방향) |
| **삽입 위치** | 뒤(Rear)에서만 가능 | 앞(Front)과 뒤(Rear) 모두 가능 |
| **삭제 위치** | 앞(Front)에서만 가능 | 앞(Front)과 뒤(Rear) 모두 가능 |
| **메커니즘** | FIFO (선입선출) | FIFO + LIFO (유연함) |
| **활용 예시** | 서비스 대기열, 맛집 예약 시스템 | 웹 브라우저의 뒤로가기/앞으로가기, 스크롤 이력 |

---

### 요약하자면
* **큐**는 먼저 들어온 데이터를 먼저 처리해야 하는 **순차적인 작업**에 최적화되어 있습니다.
* **덱**은 상황에 따라 앞뒤 어디서든 데이터를 넣고 빼야 하는 **복잡한 제어**가 필요할 때 사용합니다. 만약 큐의 기능만 필요하다면 굳이 덱을 쓰기보다 구조가 더 단순한 큐를 사용하는 것이 메모리나 설계 측면에서 더 명확할 수 있습니다.
